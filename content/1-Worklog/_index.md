This page aggregates the **12-week internship worklog**, documenting my learning journey and technical contributions as a **Cloud Serverless Architect** during the First Cloud AI Journey (FCJ) internship program.

Throughout the internship, I was responsible for researching AWS services, designing the AWS Serverless architecture, evaluating cloud solutions, estimating infrastructure costs, and preparing technical documentation for the **Smart Attendance SaaS** project, a multi-tenant attendance management platform.

## Weekly task summaries:

* **Week 1:** [AWS Cloud Fundamentals & Project Introduction](1.1-week1/) – Learned AWS Cloud fundamentals, explored Serverless Computing, studied the Smart Attendance SaaS project requirements, and identified core AWS services for the proposed solution.

* **Week 2:** [Requirement Analysis & AWS Service Research](1.2-week2/) – Analyzed business requirements, evaluated AWS services, and selected suitable cloud components for the Smart Attendance SaaS architecture.

* **Week 3:** [AWS Serverless Architecture Design](1.3-week3/) – Designed the initial cloud architecture using Amazon Route 53, AWS WAF, Amazon CloudFront, Amazon S3, Amazon Cognito, API Gateway, AWS Lambda, and Amazon DynamoDB.

* **Week 4:** [Authentication & Multi-Tenant Design](1.4-week4/) – Designed authentication and authorization using Amazon Cognito with JWT tokens, and planned secure multi-tenant access control.

* **Week 5:** [Backend Services & Data Layer](1.5-week5/) – Designed RESTful APIs with Amazon API Gateway and AWS Lambda, planned the DynamoDB data model, and optimized CRUD operations for attendance data.

* **Week 6:** [Event-Driven Architecture](1.6-week6/) – Designed asynchronous workflows using Amazon EventBridge, Amazon SQS, Amazon SNS, and AWS Lambda for notifications and background processing.

* **Week 7:** [Attendance Validation with AWS Location Service](1.7-week7/) – Designed employee check-in/check-out validation using AWS Location Service, including geofencing and location verification.

* **Week 8:** [Workflow Automation & Report Generation](1.8-week8/) – Designed report generation workflows with AWS Step Functions, storing reports in Amazon S3 and sending notifications through Amazon SNS and Amazon SES.

* **Week 9:** [Security, Monitoring & High Availability](1.9-week9/) – Improved system security using IAM, AWS WAF, CloudWatch monitoring, and applied AWS Well-Architected best practices.

* **Week 10:** [AWS Pricing Calculator & Cost Optimization](1.10-week10/) – Estimated infrastructure costs using AWS Pricing Calculator and optimized the architecture for cost efficiency.

* **Week 11:** [Architecture Refinement & Technical Documentation](1.11-week11/) – Refined the AWS architecture diagram, optimized service interactions, and completed the technical documentation.

* **Week 12:** [Final Presentation & Internship Report](1.12-week12/) – Finalized the architecture, completed the internship report, and prepared the final technical presentation.
